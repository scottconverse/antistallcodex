Codex Desktop Stop-hook gate

AntiStallCodex

A global Codex Desktop skill and Stop hook that keeps approved autonomous work moving until it is done, blocked, or waiting on a real answer.

Get the repo Install steps
codex.desktop
start AntistallON: finish this task before stopping.
allowed stop Done / Blocked / Question
turn off AntistallOFF

What it stops

The polite progress summary that quietly abandons the sprint.

Harder than a memory note

AntiStallCodex uses Codex's lifecycle hook system instead of relying on the model to remember a rule while context gets crowded.

Simple desktop command

Users arm it with one phrase: AntistallON. Turning it off is just as plain: AntistallOFF.

Scoped by project

The hook installs globally, but it only acts when a sprint is armed for the current project directory.

Stop-hook flow

Stopping becomes a deliberate ticket, not a drift.

Without the gate

  • Codex summarizes work while unfinished tasks remain.
  • The session stops at a convenient boundary.
  • The human returns to a half-finished job.

With AntiStallCodex

  • The Stop hook checks for an armed sprint.
  • No ticket means Codex gets a continuation prompt.
  • Legitimate stops are logged as Done, Blocked, or Question.

Desktop ritual

Two words for the part you actually touch.

AntistallON

Arm a sprint with the specific outcome Codex should reach before stopping.

active sprint
Done

Allow a clean stop when all authorized work is complete and verified.

disarms
Blocked

Allow a stop only when a human-only decision, credential, or external system prevents useful progress.

stays explicit
AntistallOFF

Turn the gate off for the current project whenever you want normal stopping behavior.

off switch
